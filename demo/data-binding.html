Data Binding

This page demonstrates the functionality of the XML model-view-controller.

 
 

Example 1: Simple (Control → Bind → Node → Data)

Two input controls are bound to two different elements with two different bindings. The control data and the instance data are synchronized incrementally.
Demo:
<data xmlns=""><a>Change this text.</a><b>Change this text.</b></data>

Example 2: Multiple Nodes 1 (Control1 → Bind1 → Node3 → Data1)

An input control is bound to multiple elements. In this exampe, the XPath binding expression xrx:instance('i-2')/data/* evaluates to three element nodes. The model-view-controller takes the first element and ignores the others. See the next example for a alternative repeat control.

Example 5: Multiple Bindings (Control3 ←→ Binding3 ←→ Node1 ←→ Data1)

Three different controls using three different bindings reference one element. One output control and two input controls are bound to one and the same element, but with three different XPath expressions. If one changes the value of an input control, the model-view-controller updates the instance as well as the controls bound to the same element.
